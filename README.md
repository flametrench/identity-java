# flametrench-identity (Java)

Java SDK for the [Flametrench v0.1](https://github.com/flametrench/spec) identity specification — currently scoped to the cross-language Argon2id password-hashing primitive plus its conformance fixture. The full credential / session API is on the roadmap; tests gate the load-bearing parity contract that callers rely on.

```java
import dev.flametrench.identity.PasswordHashing;

String phc = PasswordHashing.hash("correcthorsebatterystaple");
// "$argon2id$v=19$m=19456,t=2,p=1$..."

boolean ok = PasswordHashing.verify(phc, "correcthorsebatterystaple");
// true
```

## Cross-language parity

The Argon2id fixture in [the spec corpus](https://github.com/flametrench/spec/blob/main/conformance/fixtures/identity/argon2id.json) is a real PHC-encoded hash generated by Node's `argon2` package at the spec floor (m=19456 KiB, t=2, p=1). Java's `argon2-jvm` wraps libargon2 (the reference C implementation), same as Python's `argon2-cffi` and Node's `argon2`, so the fixture MUST verify identically. If `mvn test` ever fails this contract, password-based auth has become non-portable across SDKs.

## Installation

Maven:

```xml
<dependency>
    <groupId>dev.flametrench</groupId>
    <artifactId>identity</artifactId>
    <version>0.1.0</version>
</dependency>
```

Requires Java 17+. Depends on `dev.flametrench:ids` for `usr_/cred_/ses_` id generation and `de.mkammerer:argon2-jvm` for libargon2.
